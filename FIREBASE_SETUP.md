# クラウド同期のセットアップ（会社で在庫・現場を共有）

在庫と現場を、会社のみんなのスマホでリアルタイムに共有するための設定です。
**会社で1人（管理者）が最初に1回だけ**この設定をすれば、2人目以降は招待QR/リンクを開くだけで使えます。

料金：小さな会社の利用なら **Firebase の無料枠（Spark プラン）で十分**おさまります（クレジットカード登録も不要）。

---

## 管理者がやること（初回のみ・10分ほど）

### 1. Firebase プロジェクトを作る
1. https://console.firebase.google.com/ をGoogleアカウントで開く
2. 「プロジェクトを追加」→ 名前は何でもOK（例: `kiso-formwork`）
3. Googleアナリティクスは「無効」でOK →「プロジェクトを作成」

### 2. ウェブアプリを登録して「設定」を取得
1. プロジェクトのトップで、`</>`（ウェブ）アイコンをクリック
2. アプリのニックネーム（例: `kiso`）を入れて「アプリを登録」
3. 表示される `const firebaseConfig = { ... };` の**かたまりをすべてコピー**（あとでアプリに貼り付けます）

### 3. Firestore データベースを有効にする
1. 左メニュー「構築」→「Firestore Database」→「データベースの作成」
2. ロケーションは `asia-northeast1`（東京）など近い場所を選択
3. モードは「**本番環境モード**」で開始 →「有効にする」

### 4. セキュリティルールを設定する
1. Firestore Database の「ルール」タブを開く
2. 中身を下記にまるごと置き換えて「公開」

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /companies/{code} {
      allow read, write: if request.auth != null;
    }
  }
}
```

### 5. 匿名ログインを有効にする
1. 左メニュー「構築」→「Authentication」→「始める」
2. 「Sign-in method」タブ →「匿名」を選んで「有効にする」→保存

> これで「アプリを通した人だけがアクセスでき、会社コードを知っている人だけが自分の会社のデータを見られる」状態になります。

### 6. アプリに設定を入れて接続
1. アプリ（https://maui-ken.github.io/kiso-formwork/ ）の「**同期**」タブを開く
2. 「① Firebase設定」に、手順2でコピーした `firebaseConfig` を貼り付け
3. 「② 会社コード（合言葉）」に、**推測されにくい長めの言葉**を入力
   （例: `yamada-kensetsu-2024-9x7q`）。これが会社の合言葉になります
4. 「接続する」→「オンライン同期中」になれば完了

---

## 2人目以降（従業員）がやること

管理者の端末の「同期」タブに出る **招待QR** をスマホのカメラで読む、または **招待リンク** を送ってもらって開くだけ。
Firebase設定や会社コードの入力は不要で、自動的に同じ在庫・現場につながります。

そのあと「ホーム画面に追加」しておくと、アイコンから起動できて便利です。

---

## よくある質問

- **オフラインでも使える？** はい。電波が無くても最後に同期した在庫・現場を閲覧・編集でき、電波が戻ると自動で同期します。
- **同時に編集したら？** 基本は「後から保存したもの」が反映されます（最後の書き込み優先）。在庫の棚卸しなど、同時編集が少ない使い方を想定しています。
- **会社コードを変えたい／漏れたら？** 新しいコードで接続し直し、全員に新しい招待QRを配ってください。古いコードのデータは残りますが、コードを知らない人は開けません。
- **無料枠を超える？** 使用量は Firebase コンソールで確認できます。数十人規模・1日数百回の読み書き程度なら無料枠内です。
