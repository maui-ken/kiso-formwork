# 基礎型枠 割付計算（PWA）

住宅基礎工事の型枠を、**各辺の長さ**と**手持ちの型枠在庫**から自動で割り付けるスマホ用アプリです。
依存ライブラリなし（HTML/CSS/JS のみ）。スマホのブラウザで開いて「ホーム画面に追加」すればアプリとして使えます（オフライン対応）。

## 機能（v2.0）
- **各辺の長さ＋角（出隅/入隅）を手入力** — L字・コの字などカクカクした平面に対応
  - 時計回りに入力し、辺ごとの「出/入」ボタンで次の角を切り替え
  - **簡易平面図プレビュー**で形が閉じるか（出隅−入隅=4、寸法誤差）を即時チェック
- **在庫の登録・管理**（5種類）
  - ヒラパネル（1800・900など横幅別）／出隅コーナー枠／入隅コーナー枠／スライドパネル（伸縮・板）／**単管（長さ別）**
  - コーナー枠は**左右で脚長が違うもの（例 150×235）**も登録可。**複数サイズの所有**も可。向き（回転）と使う枠は「割り切れる」ように自動選択
  - **型枠の高さごとに在庫登録**可（例: 900×H600 と 1800×H300 を別在庫）。現場に「型枠の高さ」を指定すると、その高さの在庫だけで割付・過不足判定（空欄なら高さを区別しない）
  - **ヒラパネルの使用優先順位を選べる**（▲▼で並べ替え。既定は大きい順、「大きい順に並べる」ボタンで戻せる）
  - 登録した在庫は**後から編集**できる（✎ボタンで幅・高さ・脚・枚数などを修正。並び順＝優先順位は保持）
- **割付計算**（職人の手順を再現）
  1. 各角にコーナー枠を配置（外面の出隅＝出隅枠、同じ角の内面＝入隅枠）。左右非対称の枠は**向きと種類を、辺が割り切れるように最適化**して選ぶ
  2. 残り長さを **①割り切り優先（端数最小）→ ②使う枚数を最小 → ③使うサイズの種類を最小 → ④優先順位** の順でパネルに割り振る（枚数は増やさず、同じ枚数なら種類を減らし、なお同じなら優先順位で選ぶ）
  3. どうしても割り切れない端数だけを**スライドパネル**または**板（任意長カット）**で埋める（無理に割り切らず「未処理」として表示）
- **単管の必要本数計算** — 段数を現場ごとに入力（基本1段）。重ね代なしで隣に並べる前提で、辺ごとに大きい長さから組み合わせて算出
- **両面（内・外）自動計算**（内側の面は隣接する出隅で−基礎幅、入隅で＋基礎幅）
- **必要数量 vs 在庫の過不足判定**（型枠・単管とも不足数・板カット寸法を表示）
- **複数現場の管理**（現場をリストで切替・追加・削除）
- **クラウド同期（会社で共有）** — 会社共通の合言葉（コード）でつなぐと、**在庫と現場が全員のスマホでリアルタイム共有**。オフラインでも閲覧でき、電波が戻ると自動同期
  - Firebase（無料枠）を使用。初回のみ管理者が設定（[FIREBASE_SETUP.md](FIREBASE_SETUP.md)）、2人目以降は招待QR/リンクを開くだけ
  - 未設定でも従来どおり端末内（localStorage）だけで動作
- データは端末内（localStorage）に保存（同期ONなら加えてクラウドにも保存）

## 計算の前提（v2.0の割り切り）
- 単位は **mm**。すべての角は **90°** 前提。
- コーナー枠は「脚の長さ」を各辺から消費する1ユニットとしてモデル化（角の納まりは現場で微調整前提）。
- 単管は辺ごとに独立して割付（角をまたがない）。端の余りは所有する最小の1本で仕上げる。
- パネルの割付は **割り切り優先の探索**（部分和で端数を最小化 → 優先順位を尊重して分解）。1辺ごとに独立して計算し、在庫上限は考慮せず必要数を出して過不足判定（従来どおり）。

## 使い方
1. **在庫**タブで、手持ちの枠（横幅mm・枚数）と単管（長さmm・本数）を登録。
2. **図面・寸法**タブで、現場名・基礎幅・単管の段数・各辺の長さと角（出/入）を入力。
3. 「割付を計算する」→ **計算結果**タブで必要枚数・単管本数と過不足を確認。

## ローカルで動かす
```bash
cd kiso-formwork
python3 -m http.server 4173
# ブラウザで http://localhost:4173 を開く
```

## テスト（計算エンジン）
macOS の JXA で実行：
```bash
osascript -l JavaScript test_engine.js
```

## 公開（リリース）方法
静的サイトなので、以下のいずれかにフォルダを上げるだけで公開できます（PWAはHTTPS必須）:
- **Netlify**: フォルダをドラッグ＆ドロップ
- **GitHub Pages**: リポジトリにpushして Pages を有効化
- **Vercel / Cloudflare Pages**: フォルダを連携

## ファイル構成
| ファイル | 役割 |
|---|---|
| `index.html` | 画面 |
| `styles.css` | スタイル |
| `engine.js` | 割付計算エンジン（純粋関数・テスト対象） |
| `app.js` | 状態管理・UI・永続化 |
| `manifest.webmanifest`, `sw.js`, `icon.svg` | PWA（インストール・オフライン） |
| `test_engine.js` | エンジンの自動テスト |
